Веб-форма

Компонент содержит в себе вспомогательный функционал по работе с формами.

Форма описывается в файле шаблона, который сначала обрабатывается стандартным шаблонизатором, а затем специальным парсером расширенных значений, который обрабатывает дополнительные теги и атрибуты.

Данные формы отправляются только методом POST.

КОДИРОВКИ

Внутренней кодировкой форм является UTF-8. Если надо передавать данные и получать HTML в другой кодировке, её следует указать вторым аргументом в конструкторе.

ПРАВИЛА СОСТАВЛЕНИЯ ШАБЛОНОВ

  1. Шаблон должен содержать только один корневой элемент
  2. Корневым элементом должен быть элемент form
  3. В корневом элементе должно быть объявлено пространство имён "http://eresus.ru/schema/form/"
  4. У корневого элемента должен быть установлен id
  5. Атрибут method у корневого элемента необязателен и всегда будет "post"
  6. Разметка должна использовать XHTML и быть валидной
  1.    <form xmlns:fc="http://eresus.ru/schema/form/" id="form_id" ...>
  2.      ...
  3.    </form>

Все расширенные теги и атрибуты принадлежат пространству имён "http://eresus.ru/schema/form/".

  1.    <fc:tabwidget id="tabs">

  10.        <fc:tab name="tab2">...</fc:tab>

  • tabcontrol
  • tabs
Что делает тег:
  1. Создаёт "div" с таким же идентификатором и классом "tab-widget"
  2. Вставляет JS-код для инициализации плагина jQuery
tabcontrol

Переключатель вкладок.

  1.    <fc:tabcontrol>
  2.      <fc:tab name="tab1">Вкладка 1</fc:tab>
  3.      <fc:tab name="tab2">Вкладка 2</fc:tab>
  4.    </fc:tabcontrol>

Требования:

  • Может располагаться только внутри tabwidget.
Вложенные теги:
  • tab
Что делает тег:
  • Создаёт ul с переключателями вкладок
tabs

Область содержимого вкладок.

  1.    <fc:tabs>
  2.      <fc:tab name="tab1">...</fc:tab>
  3.      <fc:tab name="tab2">...</fc:tab>
  4.    </fc:tabs>

Требования:

  • Может располагаться только внутри tabwidget.
Вложенные теги:
  • tab
Что делает тег:
  • Создаёт div, содержащие все панели вкладок
tab

В зависимости от того располагается этот тег внутри tabcontrol или tab является либо кнопкой вкладки либо панелью вкладки.

  1.    <fc:tab name="tabName">Содержимое вкладки</fc:tab>

Атрибуты:

  • name - имя вкладки. Должно быть одинаковое у тега tab в tabcontrol и
соответствующего ему тега tab в tabs

Требования:

  • Может располагаться только внутри tabcontrol или tabs.
Что делает тег:
  • Создаёт li (в tabcontrol) или div (в tabs)
  • Для li значение атрибута используется для href
  • Для div значение атрибута используется для id
attach

Позволяет присоединить некоторые расширенные атрибуты ко всем элементам, удовлетворяющим jQuery-селектору.

  1.    <fc:attach to="input[name=myRadioButton]" required="true" />

В приведённом примере пользователь обязательно должен отметить одну из радио-кнопок с именем "myRadioButton".

Атрибуты:

  • to - селектор, поддерживаемый jQuery
  • required - см. описание атрибута ниже

РАСШИРЕННЫЕ АТРИБУТЫ

required

Отмечает элемент ввода как обязательный для заполнения:

  1. Добавляет в связанный с элементом <label> код '<sup>*</sup>'
  2. Добавляет JavaScript-проверку заполненности поля в обработчик submit
Возможные значения:

  • true - Проверка и вывод стандартного сообщения об ошибке
  • строка - Собственное сообщение об ошибке (пока не реализовано)
Применим к:
  • input (text, password, checkbox, radio)
  • textarea
Условия:

  • Элемент ввода должен иметь id
  1.    <input type="text" name="name" id="someId" fc:required="true" />

password

Проверяет соответствие подтверждения паролю.

Возможные значения:

  • строка - идентификатор поля пароля
Условия:

  • Применимо только к элементам input[type=password]
  • Элемент ввода должен иметь id
  • У поля пароля должен быть атрибут id
  1.    <input type="password" name="password" id="passFieldName" fc:required="true" />
  2.    <input type="password" name="confirm" id="confirmField" fc:password="passFieldName" />

match

Проверка значения поля на соответствие регулярному выражению. Внимание! Пустое значение так же считается правильным. Для проверки поля на заполненность можно комбинировать этот атрибут с атрибутом "required".

Возможные значения:

  • Регулярное выражение совместимое с JavaScript и PRCE
Условия:

  • Элемент ввода должен иметь id
  1.    <input type="text" name="name" id="someId" fc:match="/^[a-z0-9]+$/i" />
В приведённом примере будет выдано сообщение об ошибке если в поле будет введено что-либо кроме латинских букв и цифр.

email

Проверка того что в поле введён адрес email. Внимание! Пустое значение так же считается правильным. Для проверки поля на заполненность можно комбинировать этот атрибут с атрибутом "required".

Возможные значения:

  • true - Проверка и вывод стандартного сообщения об ошибке
  • строка - Собственное сообщение об ошибке (пока не реализовано)
Условия:

  • Элемент ввода должен иметь id
  1.    <input type="text" name="mail" id="someId" fc:email="true" />

ИСПОЛЬЗОВАНИЕ В PHP

Общий принцип работы с компонентом следующий:

  1. В представлении создаётся экземпляр компонента формы
  2. В форму подставляются нужные данные (при необходимости)
  3. Форма компилируется и выводится в браузер
  4. Пользователь заполняет форму и отправляет её на сервер
  5. На сервере контроллёр создаёт экземпляр компонента той же формы
  6. Считывает из неё данные и обрабатывает их
Отображение формы

  1.    # Создаём экземпляр формы
  2.    $form = new EresusForm('Common/path/form');
  3.  
  4.    # Загружаем данные в форму
  5.    $profile = new ProfileModel();
  6.    $form->setValue('profile', $profile);
  7.  
  8.    # Компилируем форму
  9.    $result = $form->compile();

Обработка пользовательского ввода

  1.      # Создаём экземпляр той же формы что и выше
  2.   $form = new EresusForm('Partner/path/form');
  3.  
  4.   $profile = new ProfileModel();
  5.   # Читаем данные
  6.   $profile->orgName = $form->getValue('orgName');
